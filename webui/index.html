Settings
Export path
Directory where log tarballs are saved after each boot.
Max files
Keeps only the N most recent logs. Older ones are deleted on boot when the limit is exceeded.
Auto-delete (days)
Log files older than this are automatically removed on boot. Set to 0 to keep files indefinitely.
Persistent logging
By default, log capture stops after the first unlock and logs are packaged immediately. Enable this to keep capturing in the background. Logs will be packaged on the next reboot instead.
Capture ramoops
Copies the previous boot's kernel crash log from pstore into the archive, if present. Useful for diagnosing boot loops and kernel panics. Has no effect on devices without pstore support.

Buffers
Main
App-level logging. Most app crashes and general debug output end up here.
System
System services like PackageManager and ActivityManager. Useful for diagnosing boot issues and service failures.
Crash
Crash traces and ANR (App Not Responding) reports. Captures full stack traces when apps or the system crash.

Stack traces may contain auth tokens or credentials leaked by apps. Review before sharing.

Radio
Telephony, modem, and RIL messages. Useful for diagnosing signal or connectivity issues.

Contains IMEI, phone numbers, and SIM identifiers. Disable before sharing logs publicly.

Events
Structured system events like battery stats, app usage, and thermal data. Mostly useful for advanced diagnostics.
Kernel
Kernel ring buffer (dmesg). Driver messages, hardware init, and low-level kernel errors. Most useful for boot loops, kernel panics, and hardware failures.

This module always captures kernel messages via /proc/kmsg into a separate kernel.log, regardless of this setting. Enabling this buffer attempts to also include kernel messages in the logcat timeline via /dev/kmsg, but many devices and custom kernels don't expose that interface, so it may produce no output.
